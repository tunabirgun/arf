# Code of Conduct

## Our standard

Arf is a small open-source project. Everyone taking part — filing issues, reviewing, or
contributing code — is expected to be respectful and constructive. In short:

- Be kind and assume good faith. Critique ideas and code, not people.
- Keep discussion on topic and useful. Explain your reasoning.
- No harassment, personal attacks, discrimination, or unwelcome behaviour of any kind.

## Scope

This applies in all project spaces — issues, pull requests, discussions, and any other
channel that represents the project.

## Enforcement

If someone's behaviour is a problem, report it to the maintainer through GitHub (open an
issue asking for private contact, or use the Security tab's private reporting for sensitive
cases). Reports will be reviewed and handled as fairly and privately as the situation allows.
The maintainer may edit or remove content, and may block accounts, that breach this standard.

This document is adapted in spirit from the
[Contributor Covenant](https://www.contributor-covenant.org/), version 2.1.
